# Leasing | Ziprent Wiki

## Overview

**Leasing: The "Contract Kings"**

The Leasing Teams (Applications, Leasing, Renewals) create a smooth move-in experience for new and existing tenants. They develop the legal contract (Lease Agreement) between Ziprent, owners, and tenants.

### Core Function
- Generate legally binding lease agreements
- Minimize risk through attention to detail
- Ensure all property notes and stipulations included
- Bridge communication between owner and tenant

### Goals

- Follow all Fair Housing requirements
- Double-check all property information
- Ensure all lease details clearly outlined (start date, utilities, HOA, rent, deposit, etc.)
- Prevent surprises that harm tenant experience
- Protect Ziprent from liability

### Staff Performance Expectations

- **Lease Request**: Owner approval within 24 hours or follow up
- **Tenant Signature**: Within 24 hours of lease issuance
- **Awaiting SD**: Daily check-in if pending
- **Response Time**: Within 1 hour on active threads

## Lease Timeline

**Start to Finish**

1. Owner accepts applicant
2. Send Lease Request to applicant (non-binding)
3. Send Lease to owner for approval
4. Applicant accepts Lease Request terms
5. Lease sent to tenants for signature
6. Tenants sign lease
7. Tenants initiate security deposit
8. Ziprent countersigns (once deposit initiated)
9. Move-in instructions sent to tenant
10. First month's rent collected (PM tenants)
11. Tenant receives lockbox code at move-in

## Lease Status Definitions

- **Lease Request**: Tenant sent terms to accept (non-binding offer)
- **Lease Approval**: Waiting for owner approval
- **Pending Lease Signing**: Waiting for tenant signature
- **Pending Deposit**: Tenant signed but not initiated deposit
- **Pending Manager Signature**: Ziprent waiting to countersign
- **Leased**: Tenant, owner, and Ziprent all signed; deposit received

## How-To: Creating a Lease Request

**Process**
1. Go to property Leases → Add Lease Request
2. Select approved tenants from Applications
3. Fill in material terms:
   - Start Date / End Date
   - Monthly Rent
   - Security Deposit
   - Grace Period (usually stays same)
4. **Important Considerations**:
   - If applicant has pet: Add $500 to security deposit
   - If property charges utilities: Add to Monthly Rent amount
   - End date calculated automatically (usually last day of month before anniversary)

**Example**: If lease starts 8/15, end date will be 7/31

## How-To: Lease to Owner

**Process**
1. Lease Request created automatically generates Docusign review
2. Check all lease parameters entered
3. Select if property under special rent control
4. Press Finish in Ziprent
5. System generates owner approval email and tenant copy email
6. Owner reviews and approves lease
7. Applicants accept Lease Request
8. **Important**: Merge Proposed Tenant Email with Lease Request Email

## Editing Leases

**Policy**: Do not edit base lease

**Instead**: Use Lease Addendum Section to add specific terms

**Examples**: Fireplace clause, special conditions, amendments

## Tenant Accepts Lease Request

**Triggers**
- Sends full lease to approved tenants for signature via DocuSign
- Owner receives notification
- Tenants sent lease signing instructions

## Tenant Sent Lease to Sign

**Approval Button**
- Sends lease to selected tenants for DocuSign signature
- Ensures signature order and placement

## Ziprent Counter Sign of Lease

**Requirement**: All tenants signed + Security deposit initiated ($)

**Process**
1. Check for ($) next to tenant who set up bank account
2. Verify all signatures received
3. Stop listing via "Leads and Listings" page
4. Countersign lease through Ziprent
5. System automatically updates status to Leased

**If Not Appearing**: Deposit hasn't been initiated yet

## Repair Request During Leasing

**Process**
1. Start repair in Ziprent with photo
2. Cross-link repair thread with leasing thread
3. Respond to owner and tenant that Repairs department will follow up
